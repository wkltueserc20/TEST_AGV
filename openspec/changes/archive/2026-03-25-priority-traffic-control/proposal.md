# Proposal: Priority-Based Traffic Control (基於優先級的多機交管系統)

## 問題背景 (Context)
目前 AGV 系統的避讓機制較為基礎，閒置的車輛會主動閃避有任務的車輛。然而，當「兩台都有任務」的 AGV 發生路徑衝突時，缺乏一套完善的交管與仲裁機制。這會導致兩車在路口互相卡死，或者無法順利通行。為了提升多車協作的效率並方便 Debug，我們需要一套基於優先級的交通管制系統，並在前端提供更細緻的狀態顯示。

## 目標 (Goals)
1.  **任務間的優先級交管**：當兩台執行任務中的 AGV 發生路徑衝突時，低優先級的車必須讓行（前往中繼安全點）。
2.  **ID 仲裁機制**：若兩車優先級相同，則透過 ID 大小來決定（ID 較小者優先執行閃避）。
3.  **智能衝突偵測**：擴展前方路徑的掃描範圍至 100 個點，提早發現潛在的碰撞。
4.  **自動任務恢復**：當閃避車輛發現高優先級車輛的路徑不再與原路徑重疊時，自動恢復原本的任務。
5.  **精細的狀態機**：新增 `WAITING`、`THINKING`、`YIELDING` 等狀態，讓前端能夠顯示交管過程，方便 Debug。

## 非目標 (Non-Goals)
- 本次不改動 A* 底層的路徑搜尋演算法（`planner.py` 中的 A* 核心不動），而是重複利用現有的 `find_nearest_safe_spot`。
- 本次不處理三台車以上的「死鎖 (Deadlock)」環狀複雜問題，專注解決兩車相遇的交管。

## 成功標準 (Success Criteria)
- 當高低優先級的車輛發生路徑衝突時，低優先級的車輛能正確切換為 `THINKING` -> `YIELDING`，並停在中繼安全點等待 (`WAITING`)。
- 高優先級車輛通過後，避讓的車輛能自動切換回 `PLANNING` 並繼續執行原任務。
- 前端能正確顯示對應的新狀態。
