# Proposal: Multi-AGV Yield List (多車讓路清單交管系統)

## 問題背景 (Context)
之前的版本已經實現了「兩車交管」，但只能處理一個 `yielding_to_id`。當三台車同時在十字路口相遇時，避讓中的 AGV 可能會遺失原本正在避讓的車輛 ID，或者只等到其中一台車離開就恢復任務，導致與另一台車發生碰撞。此外，頻繁的路徑衝突檢查也對後端性能造成一定壓力。

## 目標 (Goals)
1.  **實作多車讓路清單**：將單一 `yielding_to_id` 升級為 `yielding_to_ids` 集合，支援同時避讓多台車輛。
2.  **優化檢查頻率**：在 `WAITING` 狀態下，將路徑衝突檢查的頻率降至 **10 秒一次**。
3.  **縮短恢復延遲**：清單清空後的恢復冷卻時間從 15 秒縮短至 **2 秒**，提升物流效率。
4.  **確保安全點完整性**：在計算安全點時，必須同時考慮清單中所有車輛的規劃路徑。

## 非目標 (Non-Goals)
- 本次不更改路徑搜尋演算法（A*）。
- 不處理跨區域的全域死鎖，僅針對局部區域的多車交管。

## 成功標準 (Success Criteria)
- 三台執行任務的車輛在同一區域相遇時，低優先級的車輛能正確收集所有高優先級車輛的 ID 並執行避讓。
- 只有當清單中所有車輛都通過且衝突解除後，避讓車輛才會在 2 秒內恢復任務。
