# 多機差速 AGV 高精模擬器 (V8.0 Pro)

這是一個工業級的高精度多機 AGV (Automated Guided Vehicle) 模擬系統。採用 **Python FastAPI** 作為運算後端，配合 **React (TypeScript)** 作為可視化前端，實作了結合 **物理勢場 A*** 與 **Proactive One-Shot Evasion (主動式單次避讓)** 的先進導航架構。

## 🚀 核心特色

### 1. V8.0 主動意圖避讓架構
- **全路徑意圖廣播 (Intent Broadcast)**：每一台執行任務的車輛會將其「剩餘整條路徑」註冊到世界地圖中，讓其他車輛擁有「全知視角」。
- **單次觸發機制 (One-Shot Evasion)**：待命車一旦選定遠處安全點，便會鎖定狀態直到抵達，徹底消除傳統避讓中的頻繁抖動與震盪。
- **霸道任務優先**：執行任務的車輛擁有最高路權，發現避讓車在動時會主動停下死等，騰出 100% 的空間。

### 2. 非同步響應引擎 (Async Engine)
- **背景地圖重算 (Async Costmap)**：障礙物變更後的複雜代價地圖運算在背景執行，不阻塞 WebSocket 通訊與物理模擬。
- **非同步路徑規劃 (Async Pathfinding)**：A* 搜尋與避難點搜尋均在獨立線程運行，支援多機並行規劃且 UI 維持 60FPS。

### 3. 高精度物理模擬 (Industrial Physics)
- **子步長積分 (Sub-stepping)**：將每一幀時間拆解為多個 0.01s 的微小步長，確保即便在 30x 極速模擬下，也不會發生「穿牆」現象。
- **動態倍率同步**：支援 1x、10x、20x、30x 模擬加速，並具備時間補償邏輯以維持視覺平滑。
- **嚴格 RPM 限制**：鎖定 3000 RPM (600mm/s) 物理紅線，具備真實的加速度限制與動力學模型。

## 🧠 導航演算法：物理勢場 A*

本系統採用結合 A* 搜尋、人工勢場與動態視覺化監控的綜合導航方案。

1.  **物理平衡中心場**：利用 $1/dist^4$ 代價函數，路徑會根據兩側牆壁壓力自動尋找幾何中線，生成對 AGV 最友善的軌跡。
2.  **搜尋優化 (Tie-Breaking)**：在代價平手時，優先選擇預估距離較小的點，消除對座標大小的偏見，使搜尋朝向目標「波束化」。
3.  **演算法視覺化**：具備「思考雲」重播功能，以流水方式逐格展示 A* 搜尋過程，讓演算法不再是黑盒子。
4.  **外擺補償 (Swing-out)**：轉彎時自動補償內外輪差，確保 1m 車體在狹窄通道中完美過彎。

## 🛡️ 避讓系統技術指南

### 1. 排斥力場原理
任務車周邊 **2.0 公尺 (2000mm)** 的圓形區域被定義為「絕對禁區」。待命車會掃描自身是否落入任何人的禁區。

### 2. 避讓觸發流程
1.  **偵測 (5Hz)**：監控自身位置與他人預演路徑的重疊。
2.  **安全位搜尋 (BFS)**：
    - 在 **25 公尺** 範圍內尋找符合 `cost == 0` (離牆 2m) 的空位。
    - **定向優先**：優先往來車的反方向搜尋。
    - **最小位移**：避讓位移必須大於 **5.0 公尺**，確保一次性撤離。
3.  **A* 執行**：調用 A* 規劃去避難點的最優路徑並執行。

## 🎮 操作說明 (Windows)

1.  **啟動**：執行 `.\start.ps1` 一鍵啟動前後端。
2.  **左鍵點擊**：新增/選取障礙物（吸附網格）。
3.  **雙擊**：快速刪除障礙物。
4.  **右鍵點擊**：設定選中 AGV 的 **目標 B 點**。
5.  **側邊欄**：
    - 切換 **1x ~ 30x** 運行速度。
    - 即時監視座標、角度與左右輪 RPM（最高 3000）。
    - 觀察紫色 X (避難目標) 與淡紅色圓圈 (預演禁區)。

```text
│   ├── world.py        # 世界引擎：管理邊界、障礙物、路徑佔用與避難點預約

│   ├── controller.py   # 前進/倒車控制器、加速度限制與外擺補償

├── frontend/
│   ├── src/
│   │   ├── SimulatorCanvas.tsx # 高效能 Canvas 渲染、補間動畫與力場視覺化
│   │   └── App.tsx             # Fleet 管理與全域控制 UI
└── start.ps1           # 自動化啟動腳本
```
